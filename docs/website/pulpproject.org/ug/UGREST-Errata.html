Errata

An erratum is an entity in pulp representing update metadata information for a given package update. Errata usually has associations with Packages, Repo and Consumers.

Create

Create a new errata given the passed info in the metadata.

Status: partial - updateinfo xml needs to be updated appropriately for new errata

Path: /errata/

Method: PUT

Parameters:

  • id: unique identifier
  • title: human-readable title for erratum
  • description: errata description
  • version: erratum version
  • release: erratum release
  • type : type of errata (currently supports: ['bugzilla', 'enhancement', 'security'])
  • status : Erratum status information (optional)
  • updated : updated erratum date (optional)
  • issued : Erratum issued date (optional)
  • from_str : who issued the erratum (optional)
  • reboot_suggested : if errataum requires a reboot upon applying (optional)
  • references : includes bugzilla id info, CVE info for security errata etc (optional)
  • pkglist : packages effected by this erratum
  • immutable : can this erratum be modified (optional)
  • repo_defined: should this errata added as part of a repo vs custom (optional)
  • severity : erratum severity (Currently supports: [Critical, Important, Moderate, Low])
  • summary : erratum summary information
  • solution : any solution text associated to the errtum
  • rights: rights information

Authentication:

username/pass

Filters:

None

Response: 201 Created

Headers:

  • location: '/errata/test-errata/' 
    {'_id': 'test-errata',
 '_ns': 'errata',
 'description': None,
 'from_str': 'updates@example.org',

 'issued': '2010-08-05 23:03:01',
 'pkglist': [],
 'pushcount': '',
 'reboot_suggested': False,
 'references': [{'href' : 'https://bugzilla.redhat.com/show_bug.cgi?id=xxxxxx'), 
                'type' : 'bugzilla',
                'id'   : 'xxxxxx',
                'title': 'Add test info mode to desktop menu')}],

 'repo_defined': True,

 'title': 'test-rpm-23.2-4.fc13',

 'version': '1.4'}

Error: 409 Conflict if there is an existing errata with the same id

Install Errata on Consumers

  • Install errata on specific consumer Ids.
  • If no errata id specified, installs all applicable errata
  • If type specified, only installs errata of that types

Status: complete

Path: /consumers/<consumer id>/installerrata/

Method: GET

Parameters:

  • errataids : list of errata ids to perform installs
  • types: type of errata to perform install

Authentication:

username/pass or Consumer Certificate

Filters: None

Response: 200 OK 

['fedora-packager', 'zsh']

Error: None
