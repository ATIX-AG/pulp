Pulp includes a user model to allow access control to specific API methods and the objects stored within Pulp. The goal is to allow you to specify specific users the right or denial to do things such as:

  • create repos
  • modify existing repos
  • add packages to repos

Eventually all API methods with have a specification on which level of access is required to call them.

Base URL Path: /users/

Create

Create a consumer group

Status: DRAFT

Path: /users/

Method: PUT

Parameters:

  • id: optional user id
  • login: required unique login used to identify this user in human readable cases (CLI, webui)
  • password: optional Optional password to specificy. If none is specified user must authenticate with an x509 certificate.
  • name: optional A human readable name to use for display purposes.
  • cert: optional Optional x509 certificate identifying this user. QUESTION: Do we really need to store this?

Authentication:

username/pass

Filters:

None

Response: 201 Created

Headers: 

[{'description': 'test consumer group',
  'id': 'mygroup',
  'consumerids': []}]

Error: 409 Conflict if there is an existing user with the same id
