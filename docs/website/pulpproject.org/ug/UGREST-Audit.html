Pulp

Auditing Events

  1. Event List

Pulp supports security auditing on a number of its API calls by recording the associated events in its database. This webservices API allows read-only access to those events.

Event List

List all events

Status: complete

Path: /events/

Method: GET

Parameters:

None

Authentication:

username/pass

Filters:

  • principal (str) - the caller of an api method
  • api (str) - the api name
  • method (str) - the method name
  • field (str) - which fields are returned for each event
  • limit (int) - limit the number of events returned
  • show=errors_only - only show events that have a traceback associated with them

Response: 200 OK
/events/?limit=20&field=timestamp&field=api&field=method
