The first way to interact with the system is to create a repository. Repositories will hold the packages you add to the system.

Base URL Path: /repositories/

List

List repositories. With no filters, this will return a list of all repositories on the system. The list is empty if no repositories are found.

Status: complete

Path: /repositories/

Method: GET

Parameters:

None

Authentication:

username/pass or Consumer Certificate

Filters: (multiple filters can be supplied)

  • 'id'
  • 'name'
  • 'arch'
  • 'groupid'
  • 'relative_path'

Response: 200 OK 

[{'arch': 'x86_64',
  'feed': 'http://feed.me/',
  'id': 'my-repo',
  'name': 'my-repo'},
 {'arch': 'x86_64',
  'feed': 'http://feed.me.too/',
  'id': 'my-repo2',
  'name': 'my-repo2'}]

Error: None

Create

Create a new repository given the passed in metadata.

Status: partial - location header not currently supplied

Path: /repositories/

Method: PUT

Parameters:

  • id: unique identifier
  • name: human-readable name
  • arch: architecture
  • feed: sync source. Requires a format of the following:
    • Yum repository: yum:http://mirror.example.edu/fedora/linux/releases/12/Fedora/i386/os/
    • Local repository: yum:file://mnt/repositories/F-11/i386
    • Local directory: local:file://mnt/packages/
    • Red Hat Network: rhn:https://satellite.rhn.redhat.com/rhel-i386-server-vt-5
  • sync_schedule: crontab schedule for sync
  • cert_data: {"ca" : cacert, "cert": cert, "key" : key}
  • groupid: Group a repo belongs to eg: dev, qa, production

Take note of the 2 parts of the feed URL <type>:<path>. Supported types are:

  • yum : yum repository with existing metadata
  • local : locally mounted directory with packages
  • rhn : Connect to the Red Hat Network to get packages

Authentication:

username/pass

Filters:

None

Response: 201 Created

Headers:

  • location: '/repositories/<repository id>/' 
{'arch': 'x86_64',
 'comps_xml_path': '',
 'id': 'new-repo',
 'name': 'New Repository',
 'packagegroupcategories': {},
 'packagegroups': {},
 'packages': {},
 'source': {'supported_types': ['yum', 'local', 'rhn'],
             'type': 'yum',
             'url': 'http://feed.me/'},
 'sync_schedule': None}

Error: 409 Conflict if there is an existing repository with the same id
