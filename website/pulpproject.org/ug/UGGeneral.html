Pulp General Reference

ID Restrictions

For many entities in Pulp, an ID is used to uniquely identify one instance from another. These IDs are later used in CLI commands and URLs. As such, there are some restrictions on their values. The following are valid characters in an ID:

  • Letters
  • Numbers
  • Underscore _
  • Hyphen -
  • Period .

Time Units

Time units, like those used in timeout arguments, allow for the following units:

  • weeks
  • days
  • hours
  • minutes
  • seconds

The values are specified in the format units:length, where units is one of the above time units (case insensitive) and length is an integer greater than 1. For example: 

$ pulp-admin repo sync --id pulp-f14-64 --timeout minutes:20

Date Units

Dates are specified to both pulp-admin and pulp-client using a yyyy-mm-dd format. For example: 

$ pulp-client consumer history --start_date 2010-01-01
