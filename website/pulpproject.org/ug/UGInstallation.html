Prerequisites

  • Pulp does not currently work with SELinux, so it needs to be set to Permissive or Disabled.
  • The following ports must be open into the server:
    • 443 for API and package retrieval
    • 5672 for non-ssl or 5674 for SSL message bus connections
  • The following ports must be open into consumer systems:
    • 5672 for non-ssl or 5674 for SSL message bus connections

Server

1. Download the repository configuration file to /etc/yum.repos.d: 

$ wget /etc/yum.repos.d/fedora-pulp.repo http://repos.fedorapeople.org/repos/pulp/pulp/fedora-pulp.repo

2. Install Pulp and its dependencies: 

$ yum install pulp

3. Update /etc/pulp/pulp.conf to change the base_url attribute to reflect the hostname of the Pulp server.

4. Initialize Pulp, which starts and configures database: 

$ service pulp-server init

5. Start Pulp, which starts the database, message bus broker, and web server: 

$ service pulp-server start

6. (optional) Configure Pulp to automatically start system startup: 

$ chkconfig pulp-server on

Installation Notes

  • The default user is admin with the password admin.

Admin Client

The Pulp Admin Client is used for administrative commands on the Pulp server, such as the manipulation of repositories and content. The Pulp Admin Client can be run on any machine that can access the Pulp server, including the server itself. It is not a requirement that the admin client be run on a machine that is configured as a Pulp consumer.

Pulp admin commands are accessed through the pulp-admin script.

1. Download the repository configuration file to /etc/yum.repos.d: 

$ wget /etc/yum.repos.d/fedora-pulp.repo http://repos.fedorapeople.org/repos/pulp/pulp/fedora-pulp.repo

2. Install the Pulp client packages: 

$ yum install pulp-client

3. Update the client configuration to point to the Pulp server. Keep in mind that because of the SSL verification, this should be the fully qualified name of the server, even if it is the same machine (localhost will not work with the default apache generated SSL certificate). For more information see the UGFAQ page. This change must be done in the following places:

  • host 
$ vi /etc/pulp/client.conf

Consumer Client

The Pulp Consumer Client is present on all systems that wish to act as a consumer of a Pulp server. The Pulp Consumer Client provides the means for a system to register and configure itself with a Pulp server. Additionally, the Pulp Consumer Client runs an agent that will receive messages and commands from the Pulp server.

Pulp consumer commands are accessed through the pulp-client script. This script must be run as root to permit access to add references to the Pulp server's repositories. More information can be found in the Pulp Client User Guide.

1. Download the repository configuration file to /etc/yum.repos.d: 

$ wget /etc/yum.repos.d/fedora-pulp.repo http://repos.fedorapeople.org/repos/pulp/pulp/fedora-pulp.repo

2. Install the Pulp client packages: 

$ yum install pulp-client

3. Update the client configuration to point to the Pulp server. Keep in mind that because of the SSL verification, this should be the fully qualified name of the server, even if it is the same machine (localhost will not work with the default apache generated SSL certificate). For more information see the UGFAQ page. This change must be done in four places:

  • host
  • baseurl
  • keyurl
  • ksurl 
$ vi /etc/pulp/client.conf

4. Update /etc/gofer/agent.conf to point the agent at the broker running on the pulp server. Replace localhost in the url property with the hostname or IP of the pulp server as needed. 

$ vi /etc/gofer/agent.conf

5. Start the agent (it runs within goferd): 

$ service goferd start
